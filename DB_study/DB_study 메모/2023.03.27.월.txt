데이터베이스
:데이터를 공동으로 사용하기 위한 데이터들의 모음
데이터베이스를 관리하는 소프트웨어(s/w) 필요: 오라클
오라클을 쓸 수 있게 해주는 것: SQLDeveloper
모든 응용프로그램은 데이터베이스랑 연결된다.

데이터베이스의 종류 : 
-SQL(Structure Query Language) 구조적 질의 언어
관계형 데이터베이스 (ppt7 그림 참고)
표처럼 데이터를 관리해준다
예) 오라클(유료), mySQL(유료), msSQL(ms사 mySQL), 마리아DB(mySQL 무료버전), SQL Lite(앱DB)
-NoSQL
json모습으로 데이터를 관리해준다
SQL 문법을 사용하지 않는다
예) 몽고DB
cf) 표준(ansi) SQL - 모든 SQL문에서 사용할 수 있다
->팀별프로젝트 완성 후 서버 배포(구글,아마존,네이버클라우드,yes24서버 사용)

테이블을 만들때 컬럼 안에 들어가는 자료에 따라서 자료형을 정해야한다
char(10): 고정형 크기. 10바이트 안에 3바이트만 들어감->7바이트는 공백 . 공간낭비가 많아서 잘 안쓴다
varchar2(2): 가변형 크기. 10바이트 안에 3바이트 데이터 들어가면 3바이트형 바이트로 자료형이 정해짐
number(7) : 정수형 7자리
number(7,2) : 소수점 2자리의 실수형. 총 7자리(00000.00)
BLOB : 사진,영상 등 큰 데이터
CLOB : 텍스트 4GB 이상

데이터베이스의 특징
1) 무결성 : 결점이 없다(주민번호 같은 사람 존재X 등..), 오류가 없다 -> 가장 중요
2) 동시성 : 여러 사용자들이 하나의 테이블을 동시에 쓸 수 있도록 만드는 것
3) 영속성 : 동시성일 때 서로 관계가 없이 사용할 수 있는 것(내가 테이블 쓰는 것을 다른사람이 알 수 없음)

소프트웨어는 완전하지 않다는 명제 있음. 그러나 데이터베이스는 완전해야함

괄호기호 안에 쓸 수 있는 것
[생략 가능] {*, 컬럼명, ..} 
*은 모든 컬럼을 의미한다.

외래키 만들때 외래키가 없는것먼저 만들고 외래키 있는 것 만들어야함

데이터베이스에서 부르는 용어와 자바언어에서 부르는 용어가 약간 다름
엔티티=테이블=뷰=표 vs 클래스
속성(attribute)=컬럼=칸=열=항목=노드=item vs 필드=변수
튜플(tupe)=행=row=record=입출력의 단위
	=속성들의 집합
